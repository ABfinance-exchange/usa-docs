Query the user's cumulative transaction quota (daily / weekly / monthly / yearly) at the current KYC / VIP level and the next level. For transfer-out this endpoint reuses the channel withdrawal quota configuration.

:::tip
- For the ABF transfer-out flow, pass `bizCode=PAY_BANK` and `transCode=PAY_BANK_PAY_OUT`.
- The legacy values `WITHDRAW` / `WITHDRAW_APPLY` etc. are still accepted here for cross-flow quota lookups.
:::

### HTTP Request
GET `/fiat/public/channel/level-quota`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|fiat |**true** |string |Fiat currency, currently fixed as `USD` |
|bizCode |**true** |string |Business code. For transfer-out use `PAY_BANK`. Other values (`DEPOSIT`, `WITHDRAW`, `EXPRESS`) are supported for their respective flows |
|transCode |**true** |string |Transaction code. For transfer-out use `PAY_BANK_PAY_OUT`. Other values (`DEPOSIT_APPLY`, `WITHDRAW_APPLY`, `EXPRESS_APPLY`) map to their respective flows |
